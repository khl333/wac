## نظرة عامة

**WAC (Warm Audio Codec)** يضغط ملفات MP3 وAAC وFLAC وWAV إلى صيغة `.wac` الثنائية الخاصة، باستخدام محرك **IMA ADPCM رباعي البتات** مع معالج صوتي رقمي **(DSP) بتصميم نبضة مستوحى من أسلوب SPL** لإضافة وضوح وإيقاع حاد.

| الخاصية | القيمة |
|---|---|
| الصيغة | `.wac` (رمز التعريف: `WARM`) |
| نسبة الضغط | ADPCM رباعي البتات ← **4:1** |
| معدل أخذ العينات | 44,100 هرتز (ثابت) |
| معدل البيانات | ~172 كيلوبت/ثانية CBR |
| القنوات | ستيريو / مونو |
| معالجة DSP | مصمم النبضات (مغلف نبضي بأسلوب SPL) |
| الصيغ المدعومة | MP3، AAC، FLAC، WAV، WMA (عبر Windows MF) |

## حزمة التطبيقات

### `WarmStudio.exe` — واجهة الاستوديو الاحترافية
بيئة عمل صوتية بتصميم داكن متكامل تشمل:
- محلل طيف متحرك **بـ 48 شريطًا** مع نقاط ذروة ثابتة وتدرجات ألوان حسب التردد
- عدادا **VU** (القناة اليسرى / اليمنى)
- **لوحة الوسائط** لإدارة قائمة التشغيل بنقر مزدوج
- **شريط التقديم** بتصميم متدرج ومؤشر مخصص
- متابعة **تقدم عملية التحويل** بشكل لحظي
- أزرار التشغيل: السابق / تشغيل / إيقاف مؤقت / إيقاف / التالي

### `Transcoder.exe` — المحوّل من سطر الأوامر
يحوّل أي صيغة صوتية مدعومة في Windows إلى `.wac`:
```
Transcoder.exe <ملف_الإدخال.flac> <ملف_الإخراج.wac>
```
يُخرج أسطر `PROGRESS: nn` لتكاملها مع الواجهة الرسومية.

### `Player.exe` — مشغّل خفيف من سطر الأوامر
مشغل بسيط لملفات WAC:
```
Player.exe <ملف.wac>
```

## البناء من المصدر

### المتطلبات
- **g++** (MinGW-w64، تم الاختبار مع GCC 12 فأعلى)
- **JDK 11+** (javac + java)
- **Windows 7+** (يتطلب Media Foundation API)

### بناء المشروع
```bat
build.bat
```
أو يدويًا:
```bat
g++ -O2 Transcoder.cpp -o Transcoder.exe -municode -lmfplat -lmfreadwrite -lmfuuid -lole32
g++ -O2 Player.cpp     -o Player.exe     -municode -lwinmm
g++ -O2 Launcher.cpp   -o WarmStudio.exe -mwindows
javac WarmStudio.java
```

## كيف يعمل

### الضغط (IMA ADPCM رباعي البتات)
بدلًا من تخزين كل عينة صوتية كاملةً (16 بت)، يخزن WAC **الفرق** بين العينات المتتالية باستخدام حجم خطوة تكيّفي. هذا يحقق نسبة ضغط 4:1 نظيفة دون أي خوارزميات نفسية-صوتية معقدة.

### DSP — مصمّم النبضات (Transient Designer)
قبل الترميز، يعمل **كاشف مغلف النبضات** على إشارة PCM الخام:
1. يحسب **المشتقة اللحظية** للموجة الصوتية
2. يتتبع **متتبع ذروة سريع الهجوم** مقابل **متتبع مغلف بطيء الإطلاق**
3. يستخرج الفرق بينهما ← **طاقة النبضة الصافية**
4. يطبّق معامل ضرب `4.0x` بدقة عند لحظات ضرب الطبل وهجوم الصوت
5. يحدّ النتيجة عند 98% من السعة القصوى لترميز ADPCM نظيف

هذا يمنح مخرجات WAC جودة حيّة ونبضًا قويًا يقترب من مصدر FLAC الأصلي.

## تطور معالجة DSP

| الإصدار | الملف الصوتي |
|---|---|
| v1 – v6 | دفء صوتي، تعزيز الجهير، تشبّع تيب |
| v7 | ADPCM بدفء حقيقي |
| v8 – v8.4 | صوت حاد، عمق ستيريو ثلاثي الأبعاد، توافقيات متعددة النطاقات، ضغط OTT |
| v9 | شفافية FLAC (إزالة كل التلوين الصوتي) |
| **v9.1** | **الحالي: مصمّم النبضات SPL — نبضة نقية** |
